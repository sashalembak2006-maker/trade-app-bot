# Як зробити щоб ВСІ OTC пари мали ціну + payout

## Важливо (чесно)

Pocket Option **не дає** API де 100+ пар одночасно стрімлять як на біржі.
Максимум що можна:

| Джерело | Що дає | Затримка |
|---------|--------|----------|
| `updateAssets` (WebSocket) | **Усі пари вкладки** + payout + часто ціна в рядку | ~0.1–0.5 с після відкриття каталогу |
| `updateStream` | **1 пара** на графіку — точний live tick | realtime |
| DOM каталог | Ціни зі списку на екрані | коли список видимий |

Бот показує **реальні** ціни PO, без вигаданих чисел.

---

## Варіант A — ПК + Extension (зараз, без коду)

**Тобі треба один раз налаштувати ПК, потім просто тримати PO відкритим.**

1. Chrome → Pocket Option **demo-quick-high-low**
2. `chrome://extensions` → PRIME Bridge → **Reload** (v2.1.6+)
3. F5 на PO
4. Клікни на пару → відкрий **Assets** → вкладка **Currencies OTC**
5. **Не згортай Chrome.** Не перемикай вкладки. Залиш PO на екрані.
6. Зачекай **2 хвилини** — extension ротує категорії кожні 1.5 с
7. Extension popup → має бути **POST OK**
8. Telegram → закрий/відкрий Mini App

**Перевірка F12 Console на PO:**
```
[PRIME Bridge] WS assets: 80+
[PRIME Bridge] updateAssets: 50+ pairs
```

---

## Варіант B — 24/7 БЕЗ ПК (для продажу / без ноутбука)

**Це те, що ти робив учора через F12 — collector на Railway.**

1. PO demo → увійти в акаунт
2. Extension → **«PO сесія (F12 auth)»** (або F12 → WS → `42["auth",...]`)
3. Railway → project **rare-rebirth** → **prime-trade** → Variables:

```
PO_AUTH_MESSAGE=42["auth",{"session":"...","isDemo":1,"uid":...}]
PO_WS_URL=wss://api-eu.po.market/socket.io/?EIO=4&transport=websocket
PLATFORM_SYNTHETIC_FALLBACK=true
```

4. **`ШВИДКИЙ_ФІКС.command`** або Redeploy Railway
5. У логах Railway: `updateAssets: N pairs` (N > 50)

Тоді **клієнтам не потрібен ПК** — collector сам тримає WS з PO.

⚠️ Сесія PO живе **кілька годин/днів** — потім треба оновити `PO_AUTH_MESSAGE`.

---

## Чому зараз «ціна оновиться при сигналі»

- Bridge **працює частково** (payout 92% є = WS підключений)
- Для цих пар PO **ще не надіслав ціну** в `updateAssets` або каталог не був відкритий
- Після 1–2 хв з **відкритим OTC на ПК** ціни з’являться
